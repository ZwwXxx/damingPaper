# 完形填空题型 - 前端说明

> 日期：2026-03-09  
> 关联业务文档：`1-完形填空题型业务文档.md`  
> 适用项目：**daming-front**（学生刷题端）、**ruoyi-ui**（管理端组卷/题目编辑）

---

## 1. 文档范围

本文档只描述与完形填空相关的**前端逻辑与页面**，包括：

- 学生端刷题页（daming-front）中完形的展示、题号、导航与跳转。
- 管理端组卷页（ruoyi-ui）中完形题目的展示与编辑入口（具体题目编辑见题目管理页面文档）。

后端接口、数据模型、判分规则等以 `1-完形填空题型业务文档.md` 为准。

---

## 2. 学生端：刷题页（daming-front）

**主文件**：`daming-front/src/views/paper/index.vue`  
**接口**：`GET /quiz/student/paper/{paperId}`，返回 `PaperDto`（含 `paperQuestionTypeDto[*].questionDtos`）。

### 2.1 题号规则（与后端一致）

- **父题不占题序**：`questionType === 6` 的完形父题不参与题号计算，不在左侧显示题号，也不在内容区显示“4.”这类题号。
- **子题计入题序**：完形子题与普通题一起，按后端下发的 `itemOrder` 连续编号为 1、2、3…。
- 左侧“题目列表”与中间“可作答题目”一一对应，点击题号可滚动到对应题目并高亮。

### 2.2 核心数据与缓存

| 名称 | 含义 |
|------|------|
| `formData.paperQuestionTypeDto` | 按题型或按加入顺序的题目结构，来自接口。 |
| `orderedQuestions` | 可作答题目列表（**已排除完形父题**），按 `itemOrder` 排序，用于左侧题号与当前题高亮。 |
| `questionIndexByOrder` | `itemOrder -> 在 orderedQuestions 中的下标`，用于显示题号（如“第 5 题”）和跳转。 |
| `answerMap` | 以 `itemOrder` 为 key 的答案映射，子题用各自的 `itemOrder` 存取。 |

**构建缓存**（`buildQuestionCache`）：

- 遍历 `formData.paperQuestionTypeDto` 下所有 `questionDtos`。
- **跳过** `questionType === 6` 的题目（完形父题）。
- 其余题目（普通题 + 完形子题）按 `itemOrder` 排序后写入 `orderedQuestions` 和 `questionIndexByOrder`。

这样无论试卷是“按题型分组”还是“按加入顺序”，题号都连续，且只对可作答题目编号。

### 2.3 展示模式简述

- **按加入顺序**（`formData.numberMode === 2`）：  
  使用 `getFlatDisplayItems()` 得到“普通题 + 完形父题块”的扁平列表，按首个子题/自身 `itemOrder` 排序后渲染。
- **按题型分组**（`formData.numberMode === 1`）：  
  按 `paperQuestionTypeDto` 逐题型渲染；同一题型内先判断是否为完形父题，再决定渲染“普通题”还是“完形父题 + 子题”。

两种模式下，**完形父题都不显示题号**，**完形子题**都显示红色题号，且带 `id="child.itemOrder"` 以便跳转。

### 2.4 完形父题 + 子题的 DOM 结构（按题型分组时）

- **父题区域**：
  - 仅展示题干内容（富文本/Markdown）和“（完形填空）”标识。
  - 不渲染题号、不设置 `id` 用于锚点。
- **子题区域**（`getClozeChildren(parent)` 按 `clozeIndex` 排序）：
  - 每个子题一块，包含：
    - **红色题号**：`<span class="text-red-800 font-bold mr-2" :id="child.itemOrder">{{ getQuestionDisplayNumber(child) }}.</span>`
    - 分数、题干、选项（单选/多选/判断）与 `answerMap[child.itemOrder]` 绑定。

这样左侧题号点击时，`jumpTo(itemOrder)` 能通过 `document.getElementById(itemOrder)` 滚动到对应子题。

### 2.5 左侧题号导航与跳转

- 列表数据源：**按加入顺序**时为 `orderedQuestions`；**按题型分组**时为 `getAnchorQuestions(questType)`（该题型下排除 `questionType === 6` 的题目）。
- 显示数字：`getDisplayNumberByOrder(question.itemOrder)`，与内容区题号一致（基于 `questionIndexByOrder`）。
- 点击：`handleQuestionAnchorClick(question.itemOrder)`：
  - 非防作弊模式：`jumpTo(itemOrder)`，滚动到对应 DOM 并更新 `scrollCurrentQuestionOrder`，用于高亮当前题号。
  - 防作弊模式：按 `questionIndexByOrder` 切到对应题目索引并滚动到顶部。

### 2.6 答案初始化（initAnswer）

- 遍历 `paperQuestionTypeDto[*].questionDtos` 时，**跳过** `questionType === 6` 的父题，不为父题创建 `questionAnswerDtos` 项。
- 仅为普通题和完形子题初始化 `answerMap` 与提交用的 `questionAnswerDtos`，与后端“父题不生成答题记录”一致。

## 3. 管理端：组卷页（ruoyi-ui）

**主文件**：`ruoyi-ui/src/views/quiz/paper/edit.vue`  
**相关接口**：`getPaper`、`updatePaper`、`getQuestion`、`getClozeQuestion` 等（见业务文档）。

### 3.1 题号规则配置

- 表单中有 **题号规则**：`formData.numberMode`
  - `1`：按题型分组（左侧按题型分块显示题号）。
  - `2`：按加入顺序（左侧一列题号，与全局顺序一致）。
- 完形填空在两种模式下都遵循“父题不计入题序、子题计入”，该逻辑由**后端**在 `paperIdtoDto` 中重算 `itemOrder` 保证；前端只负责展示与组卷操作。

### 3.2 题目列表展示（组卷页）

- 使用 `getDisplayQuestions(questionType)` 过滤：只展示“非完形子题”的题目（即普通题 + 完形父题），子题不单独占一行。
- 这样列表里看到的是“一道完形填空（父题）”代表一整组，添加/删除时通过 `removeQuestionWithChildren` 连同子题一起删。

### 3.3 添加完形题到试卷

- 在“添加题目”弹框中选择题型为“完形填空题”并选择一道完形**父题**，确认后：
  - 调用 `getQuestion` 拿到父题，push 到当前题型 `questionDtos`；
  - 再调用 `getClozeQuestion(parentId)` 拉取该父题下所有子题，依次 push 到同一 `questionDtos`。
- 试卷保存时，后端按“父题不占 itemOrder、子题占连续 itemOrder”的规则写入/重算，学生端拉取后即得到正确题号。

## 4. 完形相关方法速查（daming-front/index.vue）

| 方法 | 作用 |
|------|------|
| `isClozeChild(question)` | 是否有 `parentId`，用于判断是否为完形子题。 |
| `getClozeChildren(parentQuestion)` | 从全卷 `questionDtos` 中筛出 `parentId === parentQuestion.id`，按 `clozeIndex` 排序。 |
| `getAnchorQuestions(questType)` | 题型下过滤掉 `questionType === 6`，只返回可作答题目（含完形子题），用于按题型分组时的左侧题号。 |
| `getFlatDisplayItems()` | 按加入顺序时的扁平展示项（普通题 + 完形父题块），完形父题用首个子题 `itemOrder` 参与排序。 |
| `getQuestionDisplayNumber(question)` | 展示用题号（1-based），优先用 `questionIndexByOrder[question.itemOrder] + 1`，保证与左侧一致。 |
| `getDisplayNumberByOrder(itemOrder)` | 左侧题号显示，同样基于 `questionIndexByOrder`。 |
| `buildQuestionCache()` | 构建 `orderedQuestions` 与 `questionIndexByOrder`，**排除** `questionType === 6`。 |
| `jumpTo(itemOrder)` | 根据 `id=itemOrder` 滚动到对应题目并高亮。 |

---

## 5. 注意事项

- **红色题号才可跳转**：内容区只有带 `class="text-red-800 font-bold"` 且带 `id="itemOrder"` 的题号会作为锚点；完形子题必须使用 `child.itemOrder` 作为 id，否则左侧点击无法定位。
- **防作弊模式**：单题切换时仍以 `orderedQuestions` 顺序为准，完形父题不会单独占一“题”，与题号规则一致。
- 若新增“按题型分组”下的其它展示形式（例如父题显示“第几组”而非完全不显示题号），需同时保证不破坏 `orderedQuestions` 与 `questionIndexByOrder` 的语义，否则左侧题号与跳转会错位。
