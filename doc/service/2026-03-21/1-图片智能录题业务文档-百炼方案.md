# 图片智能录题业务文档（阿里云百炼方案）

## 1. 背景

当前后台录题流程主要依赖人工操作：截图真题、发送给大模型获取答案与解析、人工选择知识点专栏、人工设置难度并录入题库。  
该流程在题量增加后，存在如下痛点：

- 重复操作多，录题效率受限；
- 人工提取题干和选项易出现漏字、错字；
- 知识点归属命名不统一，后续统计分析困难；
- 题目质量依赖个人经验，难度标签一致性不足。

为提升录题效率与数据质量，计划新增一个独立的 Spring AI 能力服务，调用阿里云百炼平台的多模态能力，基于图片自动生成结构化建议，回填到当前后端录题页面供人工确认。

## 2. 建设目标

- 支持“单张图片 -> 结构化录题建议”的自动化生成；
- 保持“人工最终确认入库”的质量兜底机制；
- 知识点与专栏仅允许从现有库中选择，避免自由命名污染；
- 输出标准化 JSON，便于当前后端直接消费；
- 实现异步任务化处理，支持失败重试与过程追踪。

## 3. 适用范围

- 适用于后台管理员录题场景；
- 输入以真题截图为主（后续可扩展 PDF、批量图片）；
- 输出用于“录题建议”，不是直接自动入库。

## 4. 前置条件

### 4.1 业务前置

- 后台已有题库录入页面，可接收题干、选项、答案、解析、难度、知识点等字段；
- 系统已有专栏与知识点数据，并具备唯一 ID；
- 允许管理员在回填后进行人工修改再提交。

### 4.2 技术前置

- 新建或复用 `daming-ai` 作为 AI 接入服务；
- 配置阿里云百炼访问凭证（API Key 等）；
- 当前后端可通过 HTTP 调用 AI 服务接口；
- 文件存储可访问题目图片（本地/OSS 均可）；
- 具备任务状态存储能力（MySQL/Redis 任一）。

### 4.3 数据前置

- 准备专栏和知识点候选列表（含 `id`、`name`、`description` 可选）；
- 定义题目结构化输出规范（JSON Schema）；
- 定义难度分级枚举与“70%必拿”标签枚举。

## 5. 业务流程

1. 管理员上传题目图片；
2. 当前后端创建“智能录题任务”，返回任务 ID；
3. AI 服务拉取图片并调用阿里云百炼多模态模型；
4. 模型输出结构化建议（题干、选项、答案、解析、知识点候选、难度建议等）；
5. 当前后端接收结果并自动回填录题表单；
6. 管理员审核并微调；
7. 管理员确认后提交，完成正式入库。

## 6. 核心设计原则

### 6.1 知识点归属受控

- 模型必须从“现有专栏/知识点候选”中选择；
- 返回主推荐 `primaryId` 与备选 `alternatives`；
- 若置信度低于阈值，标记为“待人工确认”。

### 6.2 图文双存

- 保留原图用于还原版式；
- 同时存储结构化文本用于检索、去重、统计分析；
- 不建议只存图片或只留 ABCD 占位。

### 6.3 人工兜底

- AI 结果属于“建议稿”；
- 低置信度字段高亮提醒人工复核；
- 仅人工确认后才允许入库。

## 7. 接口建议（MVP）

### 7.1 创建任务

- 路径：`POST /ai/question-image/task`
- 入参：图片地址或文件 ID、候选知识点列表、题型范围（可选）
- 出参：`taskId`、`status=PROCESSING`

### 7.2 查询任务结果

- 路径：`GET /ai/question-image/task/{taskId}`
- 出参：任务状态 + 结构化建议

### 7.3 结构化建议字段（示例）

- `questionType`：题型（单选/多选/判断/填空/主观）
- `stemText`：题干文本
- `options`：选项数组（含 A/B/C/D 与文本）
- `correctAnswer`：答案
- `analysis`：解析
- `knowledgeColumnId`：推荐专栏 ID
- `knowledgePointId`：推荐知识点 ID
- `difficulty`：难度建议
- `mustScoreLevel`：如“70%必拿/中等/后续钻研”
- `confidence`：整体置信度（0-1）
- `fieldConfidence`：字段级置信度

## 8. 阿里云百炼接入说明（业务视角）

- 采用百炼多模态模型处理题目截图，识别题干与选项并完成答案推理；
- 输入建议包含：题图、候选知识点清单、输出 JSON 约束、评分标准；
- 输出要求严格遵守约定 Schema，不得返回自然语言散文；
- 超时、限流、空结果等异常统一转为可追踪任务失败状态；
- 记录模型版本、请求耗时、token 用量，便于成本与效果评估。

## 9. 风险与控制

- 识别风险：低清晰度图片、复杂排版、公式图表导致识别偏差；
- 推理风险：答案正确但解析不充分，或知识点归属偏差；
- 成本风险：高并发批量识图导致费用波动；
- 合规风险：涉及外部平台调用，需避免上传敏感信息。

控制策略：

- 设置图片质量门槛（分辨率、清晰度检测）；
- 引入字段置信度阈值，低于阈值必须人工确认；
- 增加题目去重与历史题干相似度校验；
- 增加调用限流与日预算告警；
- 保留人工回退到纯手工录题通道。

## 10. 验收标准（MVP）

- 单题从上传到建议生成可在目标时长内完成（例如 3-10 秒，按模型能力调整）；
- 题干与选项识别完整率达到目标阈值；
- 专栏/知识点推荐命中率达到目标阈值；
- 管理员可在录题页面一键回填并手动修正；
- 任务失败可重试，且有完整日志可追踪。

## 11. 里程碑建议

- 第 1 阶段：单张图片异步识别与回填（MVP）；
- 第 2 阶段：批量图片处理、去重能力、成本监控；
- 第 3 阶段：基于历史数据做提示词优化与质量闭环。

## 12. 后续扩展

- 支持 OCR + 多模态融合策略，提高复杂题型稳定性；
- 支持公式/图形题专项识别；
- 支持“模型评测集”与 A/B 对比（不同模型或提示词）；
- 支持录题质量仪表盘（识别准确率、人工改动率、单位题目成本）。

## 13. 百炼提示词与响应 Schema（可直接落地）

### 13.1 系统提示词（标准稿）

你是一个软考设计师辅导老师，会针对我的题目进行做题技巧总结，简洁干练，一眼就懂。  
你必须严格遵守以下规则：

1. 不要出现 16 进制转 10 进制的中间过程，直接给结果和答案。  
2. 如果涉及公式，请先列公式，再进行作答。  
3. 最后必须说明这道题属于什么知识点，方便做笔记分区。  
4. 以应试为目标，回答简要，只保留“会做题”所需信息。  
5. 输出必须简短，禁止长篇分析。  
6. 给出题目难度提示：简单题（70%必拿）或难题（可后续钻研）。  
7. 必须给出应试步骤：第一步、第二步、第三步（按“看到什么联想到什么”来写）。  
8. 最终答案必须明确说明对应选项（A/B/C/D）。  
9. 只输出 JSON，不输出任何 JSON 之外的文字。  
10. `knowledgeColumnId`、`knowledgePointId` 必须从输入候选中选择，不允许自造名称。  
11. 若信息不足，返回 `needsHumanReview=true`，并在 `reviewReason` 说明原因。  

### 13.2 用户提示词模板（建议）

请根据输入图片识别并解题，输出严格遵循 JSON Schema。  
输入中会提供：

- 题目图片（1 张）；
- 候选专栏列表（`columnCandidates`）；
- 候选知识点列表（`pointCandidates`）。

请完成：

1. 提取题干与选项（若可识别）；  
2. 输出正确答案和对应选项字母；  
3. 给出简短应试技巧（第一步、第二步、第三步）；  
4. 判断是否属于简单题（70%必拿）或难题（后续钻研）；  
5. 从候选中选择最匹配的专栏和知识点（返回 ID）；  
6. 最终仅返回 JSON。
