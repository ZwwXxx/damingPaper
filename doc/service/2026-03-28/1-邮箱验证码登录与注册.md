# 邮箱注册与密码登录（邮箱即账号）

## 1. 概述

- **注册**：必须填写 **邮箱、昵称、密码、邮箱验证码**；后端将 `user_name` 与 `email` 均设为**规范化邮箱**（小写、trim）。邮箱验证码仅存于 Redis，规则见下文。
- **登录**：仅 **密码登录**。登录框可填 **邮箱**（新账号）或 **历史用户名**（老账号兼容）。`StudentDetailsServiceImpl` 先按 `user_name` 查，若未命中且输入含 `@`，再按 `email` 小写查询。
- **已取消**：邮箱验证码登录（无「邮箱+验证码直接登录」流程）。

## 2. 数据库

- 脚本：`daming-admin/sql/2026-03-28/1-daming_user_email.sql`（`daming_user.email` + 唯一索引）。

## 3. 配置（QQ 邮箱 SMTP）

见根 `application.yml` 的 `spring.mail`，推荐使用环境变量 `MAIL_USERNAME`、`MAIL_PASSWORD`（授权码）。

## 4. Redis（仅注册验证码）

| 用途 | Key 模式 | 说明 |
|------|-----------|------|
| 注册验证码 | `student:email:register:code:{email}` | 6 位数字，TTL 5 分钟 |
| 注册发送窗口 | `student:email:register:window:{email}` | 5 分钟内最多成功发送 3 次 |

## 5. 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/quiz/student/user/email/sendRegisterCode` | Body：`{ "email": "..." }` |
| POST | `/quiz/student/user/registry` | Body：`{ email, nickName, password, emailCode }` |
| POST | `/quiz/student/user/login` | 原有密码登录，`username` 填邮箱或老用户名 |

## 6. 前端

- `src/api/user.js`：仅 `sendEmailRegisterCode`（注册发码）。
- `src/views/login/index.vue`：单表单密码登录；注册弹窗为邮箱 → 验证码 → 昵称 → 密码。

## 7. 相关类

- `StudentEmailCaptchaService`：注册发码与验码。
- `StudentEmailAuthController`：仅 `sendRegisterCode`。
- `StudentRegisterRequest`：无 `userName` 字段。
- `MailUtil`：SMTP 发信。
